# ГОРНИЛО — Minecraft-лаунчер

## Что это
Desktop-лаунчер для закрытой группы друзей: авторизация через Microsoft/
лицензионный аккаунт Minecraft, лобби с несколькими модпак-сборками
разных версий, скачивание/обновление сборок, запуск игры локально или
подключение к постоянно работающим серверам (свой сервер под каждую
сборку).

## Стек
- Electron + React (frontend)
- Node.js (тонкий backend-слой внутри самого приложения, без внешнего
  сервера авторизации)
- @azure/msal-node — Microsoft OAuth2
- node-keytar — зашифрованное хранение токена в системном хранилище ОС
- electron-builder — сборка NSIS-инсталлятора для Windows
- electron-updater — автообновление лаунчера с приватного релиз-сервера

## Архитектурная модель (не путать с прокси-сетями между серверами)
Каждая сборка = отдельный набор модов = отдельный запуск клиента Minecraft.
НЕТ live-переключения между сборками внутри одной игровой сессии.
Пользователь выбирает сборку в лобби → лаунчер качает/проверяет файлы →
запускает JVM с этими модами → в самой игре пользователь сам решает:
одиночный мир или "Множественная игра" на сервере этой сборки
(IP автоматически прописывается в servers.dat при запуске).

### Референс
Полный рабочий HTML/CSS/JS прототип (3 экрана: логин с ASCII-пейзажем,
лобби со списком сборок, детальная панель с картой) — файл
`gornilo-launcher-v3.html`, приложен к проекту в `/design-reference/`.
Использовать его как источник истины для всех констант, а не
переизобретать дизайн заново.

### ASCII-пейзаж на экране логина
Генерируется процедурно на клиенте при каждом запуске (не статичная
картинка): звёздное небо + блочный ступенчатый силуэт местности
(случайный walk с квантованием "под блоки") + кластеры кустов/деревьев.
Алгоритм — см. функцию `buildLandscape()` в референс-файле, переносится
как есть в React-компонент.

## Фазы разработки — строго по порядку

Каждая фаза — отдельная сессия Claude Code. Не начинать следующую без
подтверждённой приёмки предыдущей.

### Фаза 1 — Каркас (Electron + React shell)
Рабочее окно, экран логина (заглушка кнопки) и лобби со списком сборок
на моковых данных, переключение экранов по клику.
**Приёмка:** `npm run start` открывает окно, оба экрана рендерятся и
переключаются, консоль чистая.

### Фаза 2 — JVM-запуск
Скачивание Forge + библиотек по манифесту Mojang, запуск java-процесса
с модами из локальной тестовой папки `packs/test-pack`, вывод логов JVM
в UI-консоль лаунчера через `child_process.spawn`.
**Приёмка:** реально запускается Minecraft с тестовым набором модов без
ручных действий вне лаунчера.

### Фаза 3 — Microsoft Auth
OAuth2 через `@azure/msal-node`: окно входа → обмен кода на токен →
цепочка Xbox Live → XSTS → Minecraft Services API → получение UUID+ник.
Токен — через `node-keytar`, зашифрован, никогда не логируется даже
в debug-режиме.
**Приёмка:** логин работает, ник верный, токен не читается в открытом
виде из файловой системы.

### Фаза 4 — Manifest-система
```json
{
  "packs": [{
    "id": "tech-magic", "name": "Tech & Magic", "version": "1.0.0",
    "description": "...", "minecraft": "1.16.5", "forgeVersion": "36.2.34",
    "manifestUrl": "...", "downloadUrl": "...", "sha256": "...",
    "fileSize": 5368709120, "serverIp": "play.gornilo.ru:25565",
    "mapUrl": "http://play.gornilo.ru:8100"
  }]
}
```
Скачивание с проверкой SHA256, дельта-обновления (сверка хешей
файлового дерева, тянуть только изменённое), прогрессбар в UI.
**Приёмка:** апдейт между двумя версиями одной сборки тянет только
диff, не архив целиком.

### Фаза 5 — Подключение к серверу + карта
Автопроставление `serverIp` из манифеста в `servers.dat` при запуске
клиента. Панель карты — `<webview>` на `mapUrl` (инстанс BlueMap),
встроена в детальную панель лобби по дизайн-референсу (fog-of-war
стиль).
**Приёмка:** сервер появляется в списке серверов игрока автоматически,
карта отображается и живая (не заглушка).

### Фаза 6 — Упаковка и раздача
`electron-builder` → NSIS-инсталлятор для Windows. Сборка **без**
цифровой подписи (осознанное решение — см. секцию "Раздача" ниже).
`electron-updater` → автообновление лаунчера с приватного релиз-сервера
(свой HTTP-эндпоинт, не GitHub Releases, если хочется приватности).

```yaml
# electron-builder.yml (пример)
appId: ru.gornilo.launcher
productName: Горнило
win:
  target: nsis
  icon: build/icon.ico
nsis:
  oneClick: false
  allowToChangeInstallationDirectory: true
  createDesktopShortcut: true
```

**Приёмка:** инсталлятор ставится на чистой Windows-машине (виртуалка
без установленного лаунчера ранее), приложение запускается и работает
после установки.

## Правила работы для Claude Code
- Не начинай следующую фазу без явного "фаза N принята, начинай N+1".
- Секреты/токены — только через переменные окружения или системное
  хранилище (`node-keytar`), никогда в коде или в git.
- Комментарии в коде — на английском, UI-тексты — на русском.
- Перед началом фазы — покажи план файлов, которые создашь/изменишь, до
  того как начнёшь писать код.
